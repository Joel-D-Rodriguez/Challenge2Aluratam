# 📚 Challenge 2 - Alura Oracle: Consumo de API y Backend Literario "Libreria Cuentos y Caminos"

### 💡 Sobre el proyecto 💡
Este desarrollo está basado en los criterios del **Challenge 2 de Alura**, donde se explora el consumo de la API **Gutendex**.
A partir de esta API, obtenemos una gran variedad de datos sobre libros, lo que nos permite trabajar con formatos JSON,
realizar el mapeo de entidades, crear bases de datos utilizando JPA y registrar la información para su consumo
según los requisitos planteados.

El proyecto incluye funcionalidades clave como:
- Recuperar libros almacenados.
- Consultar autores.
- Generar un top 10 de libros más descargados.

---

## 🌟 Funcionalidades principales 🌟
1. **Consumo de la API Gutendex** para obtener datos literarios.
2. **CRUD de libros**: Crear, leer, actualizar y eliminar registros.
3. **Manejo de datos DTOs** para una comunicación eficiente.
4. **Integración con JPA** para mapeo y persistencia de datos.
5. **Base de datos en MySQL**: Configuración y uso.
6. **Top 10 libros más descargados**: Estadísticas dinámicas.
7. **Interfaz de usuario básica (menú interactivo)**

## 🛠️ Instalación y requerimientos 🛠️
### Requisitos básicos:
- **IDE recomendado**: IntelliJ IDEA (u otro IDE compatible con Spring Boot).
- **Administrador de dependencias**: Maven.
- **Base de datos**: MySQL.

### Dependencias utilizadas:
- **Spring Boot 3.1.5**: Para la lógica del backend y consumo de API.
- **Spring JPA**: Mapeo de datos relacionales.
- **Driver MySQL**: Conexión con la base de datos.

---
